curl方式和raw方式访问api server

倪克松 2023-04-27 21:47:16
Categories： Kubernetes学习笔记 Tags：

Kubernetes中的资源本质就是一个API对象，这个对象的“期望状态”被API Server保存在ETCD中，然后提供RESTful接口用于更新这些对象。我们可以直接和API Server交互，使用“声明”的方式来管理这些资源（API对象），也可以通过kubectl这种命令行工具，或者client-go这类SDK。

我们可以在Kubernetes的官网中找到API文档：https://kubernetes.io/docs/reference/generated/kubernetes-api/v1.23/。

因为各个组件都是通过与api-server交互做的功能实现，所以如何与api-server做交互是扩展开发中的重中之重。下面我们以一个Deployment资源的增删改查操作为例，看一下如何与API Server交互。

3. 总结分析

  • kubectl get–raw这种方式和kubectl proxy+curl本质是一样的，只是不需要再开两个窗口分别执行两条命令了。只是不需要指定API Server的地址了，同样认证信息也不需要，这时默认用了kubeconfig中的连接信息。

  • 交互中api返回的信息是json文件格式的对象信息。
